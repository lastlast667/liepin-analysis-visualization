## 一、通用规范

### 1.1 请求规范

| 项目 | 说明 |
|------|------|
| 请求方法 | GET / POST（本文档接口均为 GET） |
| Content-Type | `application/json` |
| 字符编码 | UTF-8 |
| 时间格式 | ISO 8601（如 `2026-05-30T14:30:00Z`） |

### 1.2 响应规范

所有接口统一返回以下结构：

```json
{
  "code": 200,           // 业务状态码，200 表示成功
  "message": "success",  // 状态描述
  "data": { ... }        // 业务数据（具体结构见各接口）
}
```

### 1.3 全局错误码

| 错误码 | 含义 | 说明 |
|--------|------|------|
| 200 | 成功 | 请求处理成功 |
| 400 | 请求参数错误 | 参数缺失、类型错误或格式非法 |
| 401 | 未授权 | Token 缺失或已过期 |
| 403 | 禁止访问 | 无权访问该资源 |
| 404 | 资源不存在 | 岗位 ID 不存在 |
| 429 | 请求过于频繁 | 触发限流 |
| 500 | 服务器内部错误 | 服务端异常 |

## 二、接口 1：岗位列表搜索

### 2.1 接口基本信息

| 项目 | 内容 |
|------|------|
| 接口名称 | 岗位列表搜索 |
| 请求方法 | `GET` |
| 请求路径 | `/api/v1/jobs/search` |
| 接口说明 | 支持多维度筛选、排序、分页；返回列表页卡片所需字段 + 右侧全局热门统计（不受筛选条件影响） |

### 2.2 请求参数（Query 参数）

| 参数名 | 类型 | 是否必填 | 默认值 | 说明 |
|--------|------|----------|--------|------|
| `keyword` | string | 否 | `""` | 关键词，支持模糊匹配岗位标题、公司名称、岗位标签（tags） |
| `city` | string | 否 | `""` | 城市名称，精确匹配。空字符串表示"全国" |
| `salary_range` | string | 否 | `""` | 薪资范围，枚举值见[数据字典](#四数据字典)。空字符串表示"不限" |
| `education` | string | 否 | `""` | 学历要求，枚举值见[数据字典](#四数据字典)。空字符串表示"不限" |
| `experience` | string | 否 | `""` | 经验要求，枚举值见[数据字典](#四数据字典）。空字符串表示"不限" |
| `sort_by` | string | 否 | `"default"` | 排序方式，枚举：`default`（综合匹配度）、`salary`（薪资水平）、`experience`（经验要求）、`education`（学历要求） |
| `page` | integer | 否 | 1 | 当前页码，从 1 开始 |
| `page_size` | integer | 否 | 10 | 每页条数，最大 50 |

> **筛选逻辑说明**：
> - `keyword`：对 `title`、`company_name`、`company_tags` 三个字段进行模糊匹配（不区分大小写）
> - `salary_range`：后端根据 `month_salary_avg` 字段进行数值范围筛选，映射规则见数据字典
> - `experience`：后端实际匹配 `experience_level` 字段（前端展示用 `experience`，筛选用 `experience_level`）

### 2.3 响应参数

```json

  "message": "success",
  "data": {

    "page": 1,
    "page_size": 10,

      {
        "id": 5,

```

#### 2.3.1 分页字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `total` | integer | 符合筛选条件的岗位总数 |
| `page` | integer | 当前页码 |
| `page_size` | integer | 每页条数 |
| `total_pages` | integer | 总页数 |

#### 2.3.2 岗位列表字段（jobs 数组内）—— 仅返回列表页展示所需字段

| 字段 | 类型 | 说明 | 前端使用位置 |
|------|------|------|-------------|
| `id` | integer | 岗位唯一标识 | 卡片收藏、跳转详情 |
| `title` | string | 岗位标题 | 卡片主标题，可点击进入详情 |
| `company_name` | string | 公司名称 | 卡片展示 |
| `salary` | string | 薪资范围（展示用，如"35K-60K"） | 卡片展示 |
| `location_city` | string | 所在城市 | 卡片展示 |
| `education` | string | 学历要求 | 卡片展示 |
| `experience` | string | 经验要求（展示用） | 卡片展示 |
| `recruit_count` | string | 招聘人数 | 卡片展示 |
| `company_industry` | string | 公司所属行业 | 卡片展示 |
| `is_favorited` | boolean | 当前用户是否已收藏该岗位 | 卡片右上角星形按钮状态 |

> **注意**：列表接口不返回 `company_tags`、`job_url`、`job_description`、`language_requirement`、`has_weekend_off`、`month_salary_avg`、`experience_level`、`company_scale`、`update_time`、`recruit_count_parsed` 等字段。这些字段在岗位详情接口中返回。

#### 2.3.3 热门统计字段（不受筛选条件影响，基于全量数据）

| 字段 | 类型 | 说明 |
|------|------|------|
| `hot_jobs` | array | 薪资 Top 5 岗位，按 `month_salary_avg` 降序 |
| `hot_jobs[].id` | integer | 岗位 ID |
| `hot_jobs[].title` | string | 岗位标题 |
| `hot_jobs[].company_name` | string | 公司名称 |
| `hot_jobs[].salary` | string | 薪资 |
| `hot_cities` | array | 岗位数 Top 10 城市 |
| `hot_cities[].name` | string | 城市名 |
| `hot_cities[].count` | integer | 该城市岗位数量 |
| `hot_companies` | array | 岗位数 Top 5 公司 |
| `hot_companies[].name` | string | 公司名称 |
| `hot_companies[].job_count` | integer | 该公司岗位数量 |

### 2.4 请求示例

```bash
# 示例 1：搜索北京的 Java 岗位，薪资 30K 以上，按薪资排序，第 1 页

# 示例 2：无筛选，默认综合排序
GET /api/v1/jobs/search?page=1&page_size=10

# 示例 3：搜索应届生岗位，学历本科
GET /api/v1/jobs/search?education=本科&experience=应届生&page=1
```

### 2.6 排序规则说明

| `sort_by` 值 | 排序逻辑 | 排序方向 |
|-------------|----------|----------|
| `default` | 综合匹配度（关键词匹配标题 > 匹配公司名 > 匹配标签） | 降序 |
| `salary` | `month_salary_avg` 数值 | 降序（高薪在前） |
| `experience` | 经验等级顺序：应届生 < 实习生 < 1-3年 < 3-5年 < 5-10年 | 升序 |
| `education` | 学历等级顺序：博士 > 硕士 > 统招本科 > 本科 > 专科 | 降序 |

## 三、接口 2：岗位详情

### 3.1 接口基本信息

| 项目 | 内容 |
|------|------|
| 接口名称 | 岗位详情 |
| 请求方法 | `GET` |
| 请求路径 | `/api/v1/jobs/{id}` |
| 接口说明 | 获取指定岗位的完整信息，包含公司统计、相似岗位推荐、薪资分析、收藏状态 |

### 3.2 请求参数（Path 参数）

| 参数名 | 类型 | 是否必填 | 说明 |
|--------|------|----------|------|
| `id` | integer | 是 | 岗位 ID，URL 路径参数 |

### 3.3 请求头

| 请求头 | 是否必填 | 说明 |
|--------|----------|------|
| `Authorization` | 是 | `Bearer <token>`，用于获取当前用户的收藏状态 |

#### 3.4.1 岗位基本信息（job）—— 详情页完整字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | integer | 岗位 ID |
| `title` | string | 岗位标题 |
| `company_name` | string | 公司名称 |
| `salary` | string | 薪资范围 |
| `location_city` | string | 所在城市 |
| `education` | string | 学历要求 |
| `experience` | string | 经验要求 |
| `recruit_count` | string | 招聘人数 |
| `company_industry` | string | 公司所属行业 |
| `company_scale` | string | 公司规模 |
| `has_weekend_off` | boolean | `true` 显示"周末双休"，`false` 显示"弹性工作" |
| `job_url` | string | 原始招聘链接，用于"立即申请"跳转 |
| `company_tags` | array[string] | 岗位亮点/福利标签 |
| `job_description` | string | 岗位描述（完整文本） |
| `language_requirement` | string | 语言要求（空字符串表示无要求） |
| `month_salary_avg` | integer | 平均月薪（元），用于薪资分析计算 |

#### 3.4.2 公司统计（company_stats）

> 基于该 `company_name` 在平台中的所有岗位聚合计算

| 字段 | 类型 | 说明 | 计算逻辑 |
|------|------|------|----------|
| `job_count` | integer | 该公司在平台上的岗位总数 | `COUNT(*) WHERE company_name = ?` |
| `recruit_total` | integer | 该公司总招聘人数 | `SUM(recruit_count_parsed) WHERE company_name = ?` |
| `avg_salary` | string | 该公司平均薪资 | `AVG(month_salary_avg) / 1000` 保留 1 位小数 + `"k"` |

#### 3.4.3 相似岗位推荐（similar_jobs）

> 推荐策略：多源混合推荐架构。当前实现为基于同行业（`company_industry`）的 Top 3

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | integer | 岗位 ID |
| `title` | string | 岗位标题 |
| `company_name` | string | 公司名称 |
| `location_city` | string | 所在城市 |
| `salary` | string | 薪资范围 |

> **未来扩展**：接入推荐算法后，similar_jobs 将基于"当前岗位内容相似度 + 用户历史行为"混合排序，权重可配置。

#### 3.4.4 薪资分析（salary_analysis）

> 基于当前岗位所在行业（`company_industry`）的全量岗位数据计算

| 字段 | 类型 | 说明 | 计算逻辑 |
|------|------|------|----------|
| `industry_avg` | string | 行业平均薪资 | `AVG(month_salary_avg) / 1000` 保留 1 位小数 + `"k"` |
| `above_percentage` | integer | 高于/低于行业平均的百分比 | `ROUND((current - industry_avg) / industry_avg * 100)`。正值表示高于平均，负值表示低于平均 |
| `range_min` | string | 行业最低薪资 | `MIN(month_salary_avg) / 1000` 取整 + `"k"` |
| `range_max` | string | 行业最高薪资 | `MAX(month_salary_avg) / 1000` 取整 + `"k"` |
| `position_percentage` | float | 当前薪资在行业分布中的位置百分比 | `((current - min) / (max - min)) * 100`，范围 `[0, 100]` |

#### 3.4.5 收藏状态

| 字段 | 类型 | 说明 |
|------|------|------|
| `is_favorited` | boolean | 当前登录用户是否已收藏该岗位。未登录用户固定返回 `false` |

## 四、数据字典

### 4.1 薪资范围（salary_range）

| 枚举值 | 筛选逻辑（month_salary_avg，单位：元） |
|--------|----------------------------------------|
| `""`（空） | 不限 |
| `"0-10K"` | `< 10000` |
| `"10-20K"` | `>= 10000 AND < 20000` |
| `"20-30K"` | `>= 20000 AND < 30000` |
| `"30-50K"` | `>= 30000 AND < 50000` |
| `"50K+"` | `>= 50000` |

### 4.2 学历要求（education）

| 枚举值 | 说明 |
|--------|------|
| `""`（空） | 不限 |
| `"专科"` | 专科 |
| `"本科"` | 本科 |
| `"统招本科"` | 统招本科 |
| `"硕士"` | 硕士 |
| `"博士"` | 博士 |

### 4.3 经验要求（experience / experience_level）

| 枚举值（experience_level） | 展示值（experience） | 说明 |
|---------------------------|---------------------|------|
| `""`（空） | `""` | 不限 |
| `"实习生"` | `"实习生"` | 实习生 |
| `"应届生"` | `"应届"` | 应届生 |
| `"1-3年"` | `"1-3年"` | 1-3年经验 |
| `"3-5年"` | `"3-5年"` | 3-5年经验 |
| `"5-10年"` | `"5-10年"` | 5-10年经验 |

> **注意**：前端筛选时传递 `experience` 参数，后端实际匹配 `experience_level` 字段。`experience` 字段用于列表展示，`experience_level` 字段用于筛选和排序。

### 4.4 排序方式（sort_by）

| 枚举值 | 说明 | 排序字段 |
|--------|------|----------|
| `"default"` | 综合排序（匹配度） | 关键词匹配度得分 |
| `"salary"` | 薪资水平 | `month_salary_avg` DESC |
| `"experience"` | 经验要求 | `experience_level` 按等级顺序 ASC |
| `"education"` | 学历要求 | `education` 按等级顺序 DESC |

## 五、前端 Mock → 真实 API 映射对照

| 前端数据/计算 | 前端实现方式 | 真实 API 对应 |
|--------------|-------------|--------------|
| `allJobsData`（全量岗位数据） | 前端写死 18 条 Mock 数据 | 由后端数据库 `jobs_job` 表提供 |
| `cityOptions`（城市下拉选项） | 从 `allJobsData` 提取 `location_city` 去重 | 接口 1 响应中的 `hot_cities` 或后端单独提供城市字典接口 |
| `filteredJobList`（筛选+排序） | 前端 `computed` 对 `allJobsData` 过滤 | 接口 1 的 `jobs` 字段（后端完成筛选排序） |
| `paginatedJobs`（分页切片） | 前端 `slice` 分页 | 接口 1 的 `jobs` 字段（后端分页） |
| `hotJobs`（热门岗位 Top5） | 按 `month_salary_avg` 降序取前 5 | 接口 1 的 `hot_jobs` 字段 |
| `hotCities`（热门城市 Top10） | 按 `location_city` 分组计数 | 接口 1 的 `hot_cities` 字段 |
| `hotCompanies`（热门公司 Top5） | 按 `company_name` 分组计数 | 接口 1 的 `hot_companies` 字段 |
| `selectedJob`（当前选中岗位） | 点击时从列表中直接赋值 | 接口 2 的 `job` 字段 |
| `jobDetailCompanyStats`（公司统计） | 前端过滤 `allJobsData` 聚合计算 | 接口 2 的 `company_stats` 字段 |
| `similarJobs`（相似岗位 Top3） | 过滤同行业岗位取前 3 | 接口 2 的 `similar_jobs` 字段 |
| `salaryAnalysis`（薪资分析） | 前端按行业聚合计算 | 接口 2 的 `salary_analysis` 字段 |
| `companyJobsList`（公司所有岗位） | 前端过滤 `allJobsData` | 后续公司详情页接口（本文档未包含） |
| `favoriteIds`（收藏列表） | 前端 `Set` 存储 | 需后端 `user_favorites` 表 + 收藏操作接口（本文档未包含） |
| `isFavorited()` | 查 `favoriteIds` | 接口 1/2 的 `is_favorited` 字段 |
