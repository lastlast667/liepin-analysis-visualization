# 猎聘数据分析平台 — 岗位搜索模块需求文档

> **文档版本**：v1.0  
> **模块名称**：岗位搜索（Job Search）  
> **前端页面**：`JobSearchView.vue`（列表页）+ `JobDetailView.vue`（详情页）  
> **数据来源**：`jobs_job` 表 + 复用现有分析 API  
> **约束**：仅编写前端代码，不修改后端

---

## 一、模块概述

岗位搜索模块是平台的核心功能之一，允许用户通过多维度条件筛选岗位数据，以卡片列表形式展示结果，并支持进入岗位详情页查看完整信息。列表页右侧提供全局统计洞察（热门岗位、热门城市、热门公司），详情页右侧提供个性化推荐和薪资分析。

---

## 二、页面结构

### 2.1 岗位搜索列表页（JobSearchView）

#### 2.1.1 顶部搜索栏（同一行横向排列）

| 控件 | 类型 | 选项/说明 | 对应字段 |
|------|------|-----------|----------|
| 关键词输入框 | Input | 占位提示："请输入岗位名称、公司或技能" | `title` / `company_name` / 技能关键词 |
| 城市选择 | Select | 下拉选项：从现有岗位数据中动态提取所有城市 | `location_city` |
| 薪资范围 | Select | 不限、10k以下、10-20k、20-30k、30-50k、50k以上 | `month_salary_avg` |
| 学历要求 | Select | 不限、专科、本科、统招本科、硕士、博士 | `education` |
| 经验要求 | Select | 不限、实习生、应届生、1-3年、3-5年、5-10年 | `experience_level` |
| 搜索按钮 | Button | 位于最右侧，点击后触发搜索请求 | — |

- **默认值**：所有下拉框默认选中"不限"（城市默认"全国"）
- **无重置按钮**：用户需手动修改条件
- **交互**：点击搜索按钮后，携带所有筛选参数请求后端接口，刷新列表数据

#### 2.1.2 搜索结果区域

**左上角**：文本显示 `找到 x 个匹配的岗位`，`x` 为后端返回的总数

**右上角**：排序下拉框
- 综合排序（默认）：根据 `title` 和关键词匹配度排序
- 薪资水平：按 `month_salary_avg` 排序
- 经验要求：按 `experience_level` 排序
- 学历要求：按 `education` 排序

**岗位卡片列表**：
- 分页展示，每页最多 10 张卡片
- 支持翻页（页码切换或上一页/下一页）
- 单张卡片展示字段：
  - 岗位标题（`title`）——可点击，进入详情页
  - 公司名称（`company_name`）
  - 薪资（`salary`）
  - 所在城市（`location_city`）
  - 学历要求（`education`）
  - 经验要求（`experience`）
  - 招聘人数（`recruit_count`）
  - 公司行业（`company_industry`）
- 卡片右上角：星形收藏按钮（点击切换收藏/取消收藏状态）
- 鼠标悬停：卡片有轻微上浮/阴影等互动效果

#### 2.1.3 右侧统计面板（不受顶部筛选影响，始终展示全局数据）

**卡片 1：热门岗位（薪资 Top 5）**
- 排序依据：`month_salary_avg` 降序
- 展示：序号 + 岗位标题（`title`，可点击进入详情页）+ 公司名称（`company_name`）+ 薪资（`salary`）

**卡片 2：热门城市（岗位数 Top 10）**
- 展示格式：`城市名（岗位数量）`，例如 `深圳（30）`
- 交互：点击标签后，自动将顶部城市筛选下拉框设置为该城市，并触发搜索

**卡片 3：热门公司（岗位数 Top 5）**
- 数据来源：复用 `company_analysis` 的 `companies_by_jobs`
- 展示：公司名称（`company_name`，可点击进入公司详情页）+ 岗位数量

---

### 2.2 岗位详情页（JobDetailView）

#### 2.2.1 页面导航
- 左上角："返回搜索结果" 按钮，点击后返回列表页（保留之前的筛选条件和页码）

#### 2.2.2 第一排：岗位信息主卡片（大卡片）

**左侧展示字段**：
- 岗位标题（`title`）
- 公司名称（`company_name`）
- 薪资（`salary`）
- 所在城市（`location_city`）
- 学历要求（`education`）
- 经验要求（`experience`）
- 招聘人数（`recruit_count`）
- 公司行业（`company_industry`）
- 公司规模（`company_scale`）
- 是否双休（`has_weekend_off`）：`True` → 显示"周末双休"，`False` → 显示"弹性工作"
- 岗位链接（`job_url`）：用于跳转

**右侧操作区**：
- 收藏按钮（星形，点击切换状态）
- 立即申请按钮（点击后新标签页打开 `job_url`）

#### 2.2.3 第二排：岗位详情卡片

- **岗位亮点**（`company_tags`）
- **岗位描述**（`job_description`）
- **语言要求**（`language_requirement`）

#### 2.2.4 第三排：公司介绍卡片

展示字段：
- 公司名称（`company_name`）
- 所属行业（`company_industry`）
- 公司规模（`company_scale`）
- 岗位数量（复用 `company_analysis` 的 `company_by_jobs`）
- 招聘人数（复用 `company_analysis` 的 `company_by_recruit`）
- 平均薪资：计算该公司所有岗位的 `month_salary_avg` 平均值，四舍五入后格式化为 `X.Xk`（如 `8.0k`）

**底部**："查看该公司所有岗位" 文字链接，点击进入公司详情页

#### 2.2.5 详情页右侧纵向面板

**卡片 1：相似岗位推荐（Top 3）**
- 展示：岗位标题（`title`）+ 公司名称（`company_name`）+ 所在城市（`location_city`）+ 薪资（`salary`）
- 策略：多源混合推荐架构（当前岗位内容源 + 用户行为源），权重可配置
- 当前阶段：使用 Mock 数据填充，不支持点击互动
- 注意：独立岗位推荐页的推荐源配置为用户行为源；当用户行为源为空时，默认推荐 "Python"、"Java"、"前端" 等热门关键词岗位

**卡片 2：薪资分析**
- 当前岗位薪资（`salary`）
- 行业平均薪资（复用薪资分析 API，待实现）
- 高于行业平均的百分比（如 "高于行业平均 15%"）
- 行业薪资范围分布：最低薪资 → 最高薪资，并标注当前岗位薪资在分布中的位置（可视化进度条或标记点）

## 四、交互规则汇总

| 交互元素 | 行为 | 备注 |
|----------|------|------|
| 搜索按钮 | 触发筛选请求，刷新列表，重置到第 1 页 | 保留当前排序方式 |
| 排序下拉框 | 切换排序方式，重新请求数据 | 不改变筛选条件 |
| 分页切换 | 切换页码，请求对应页数据 | 保留筛选和排序条件 |
| 岗位标题（列表/热门岗位） | 跳转岗位详情页 | 需传递岗位 ID |
| 城市标签（热门城市） | 自动填充城市筛选框并触发搜索 | 快捷筛选 |
| 公司名称（热门公司） | 跳转公司详情页 | 公司详情页暂不设计 |
| 收藏星形按钮 | 切换收藏/取消收藏状态 | 需登录状态，未登录提示登录 |
| 立即申请按钮 | 新标签页打开 `job_url` | — |
| 返回搜索结果 | 返回列表页 | 恢复之前的筛选条件和页码 |
| 查看该公司所有岗位 | 进入公司详情页 | 公司详情页暂不设计 |
| 相似岗位推荐 | 当前仅展示，不支持点击 | Mock 数据阶段 |

## 五、待后端实现依赖项

以下功能需要后端接口支持，前端当前阶段使用 Mock 数据：

1. **薪资分析 API**：行业平均薪资、薪资范围分布
2. **相似岗位推荐 API**：多源混合推荐算法接口
3. **用户收藏 API**：收藏/取消收藏、查询收藏状态
4. **用户行为记录 API**：用于推荐系统的用户行为源

## 六、数据库设计建议（供后端参考）

> 注意：前端开发阶段不修改后端，但以下建议供后续迭代参考。

### 6.1 现有表
- `jobs_job`：已包含所有岗位基础字段和公司冗余字段

### 6.2 建议新增表

**`companies` 表（公司维度表）**
| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | INT PK | 主键 |
| `company_name` | VARCHAR | 公司名称（唯一） |
| `company_industry` | VARCHAR | 所属行业 |
| `company_scale` | VARCHAR | 公司规模 |
| `job_count` | INT | 岗位数量（可冗余或实时计算） |
| `avg_salary` | DECIMAL | 平均薪资 |
| `total_recruit` | INT | 总招聘人数 |
| `created_at` | DATETIME | 创建时间 |
| `updated_at` | DATETIME | 更新时间 |

**`users` 表（用户基础表）**
| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | INT PK | 主键 |
| `username` | VARCHAR | 用户名 |
| `email` | VARCHAR | 邮箱 |
| `password_hash` | VARCHAR | 密码哈希 |
| `created_at` | DATETIME | 创建时间 |

**`user_favorites` 表（用户收藏记录）**
| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | INT PK | 主键 |
| `user_id` | INT FK | 关联用户 |
| `job_id` | INT FK | 关联岗位 |
| `created_at` | DATETIME | 收藏时间 |

- 联合唯一索引：`UNIQUE(user_id, job_id)` 防止重复收藏
