# Liepin Crawler 模块生成指令集

> 本文件定义需要 GPT 生成的各个代码模块的功能要求、技术约束和接口规范。不含实现代码。

---

## 一、列表页爬取模块

### 功能要求
- 使用 requests 库请求拉勾网搜索列表页
- 解析 HTML 提取以下字段：详情页链接、岗位标题（摘要）、公司名称、薪资（摘要）、工作地点（摘要）
- 完整字段（特别是岗位标签和JD正文）不在这里提取，留给详情页模块
- 支持按搜索关键词（kd）和页码组合请求

### 输入
- 搜索关键词字符串（如"Java"）
- 页码整数（1-10）

### 输出
- 列表数据，每条记录只需要有详情页链接URL

### 技术约束
- URL 格式：`https://www.liepin.com/zhaopin/?city=410&dq=410&pubTime=&currentPage={{page}}&pageSize=40&key={{keyword}}`
- 所有请求走 config.py 配置的延迟范围（2-5秒随机）
- 请求头从 config.py 读取（含随机UA和Cookie）
- 使用 lxml 或 BeautifulSoup 解析 HTML
- 解析失败时记录 warning 日志，跳过该条不中断流程
- 职责单一：只提取详情页链接

## 二、详情页爬取模块

### 功能要求
- 使用 requests 库请求详情页
- 从详情页提取完整字段：
  - 岗位标题（title）——标注校验的依据
  - 职位描述（job_description）——ML向量化的核心输入
  - 公司名称、薪资、地点、经验、学历等辅助字段

### 输入
- 详情页 URL 列表（由列表页模块产出）

### 输出
- 完整职位数据 dict，包含所有字段

### 技术约束
- 请求间隔 3-5 秒随机
- 拉勾网返回 UTF-8，一般无需额外编码转换
- 解析失败时返回空 dict 而非抛出异常
- title 字段必须提取成功

## 三、数据标注模块

### 功能要求
- 维护搜索关键词到训练类别的默认映射表
- 维护标题关键词校验规则
- 对每条岗位数据：
  1. 根据爬取时使用的搜索关键词确定默认标签
  2. 检查标题是否与搜索关键词严重冲突
  3. 冲突时以标题为准修正标签
  4. 无冲突时保留默认标签

### 标注规则

**第一步：搜索关键词默认标注**

| 搜索关键词  | 默认 train_label |
|--------|----------------|
| Java   | Java后端开发       |
| Python | Python后端开发     |
| Go     | Go后端开发         |
| C++    | C++后端开发        |
| PHP    | PHP后端开发        |
| 爬虫     | 爬虫工程师          |
| 嵌入式    | 嵌入式开发          |
| 前端     | 前端开发           |
| 数据分析   | 数据分析师          |
| 大数据    | 大数据开发工程师       |
| 算法     | 算法工程师          |
| 软件测试   | 测试工程师          |
| 运维     | 运维工程师          |
| 全栈     | 全栈工程师          |

**第二步：标题冲突校验**

冲突判定（需修正）：
- 搜索关键词=Java，标题含 "Python" 且不含 "Java" → 修正为 Python后端开发
- 搜索关键词=Python，标题含 "Java" 且不含 "Python" → 修正为 Java后端开发
- 搜索关键词=Python，标题不含任意一个关键词 → 过滤
- 搜索关键词=算法，标题含 "测试工程师" → 修正为 测试工程师

不冲突（保留默认标签即搜索关键词）：
- 标题同时含 "Java" 和 "Go"（正常多技能）
- 标题模糊如 "后端开发工程师"（无法明确判断）
- 标题含搜索关键词的同义词

**第三步：保留默认**

标题与搜索关键词没有明显冲突 → 保留默认标签。

### 为什么 labels 不参与标注

- 标签由招聘方填写，可能存在疏忽
- 标题是招聘方最关注的表达，优先级最高
- 搜索关键词已经提供了强烈的类别信号
- labels 仅作为特征用于后续 ML 向量化

## 四、数据清洗与入库流水线

### 功能要求
- 薪资解析：将猎聘薪资字符串解析为最小值、最大值
- URL 去重：以 job_url 为唯一键
- 批量入库：每 100 条一个事务批次

### 去重规则
- 同一条岗位可能被多个关键词搜到（如标题同时含 Java 和 Python）
- 去重时以 job_url 为唯一键
- 保留字段最全的那条

### 批量入库规则
- 每 100 条一个事务
- 以 job_url 为唯一键，重复跳过
- 单条失败不影响同批其他数据

## 五、关键词调度模块

### 功能要求
- 管理搜索关键词队列（13 个关键词）
- 记录爬取进度，支持中断后断点续爬
- 每完成一个关键词，保存进度快照

### 进度快照内容
- 当前关键词索引
- 已完成的关键词列表
- 累计爬取的职位数量
- 最后更新时间

---

## 六、通用工具模块

### 功能要求
- 带重试机制的 HTTP 请求封装
- 日志配置
- Cookie 管理（首次访问拉勾首页获取，后续复用）
- 滑块验证检测（检查响应是否包含验证页面关键字）

---

## 七、模块间数据流转

```
关键词调度器 产出: keyword
          ↓
列表页爬虫 产出: 列表数据（含 job_url 和摘要信息）
          ↓
详情页爬虫 产出: 完整职位数据（含 title、labels、job_description）
          ↓
标注模块 产出: 带 train_label 的数据（搜索关键词默认 + 标题校验修正）
          ↓
数据清洗流水线 处理: 去重 → 薪资解析 → 入库
```
